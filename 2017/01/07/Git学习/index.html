Git学习系列

  1. git status查看git仓库的状态

  2. git init 初始化git仓库

  3. git add a.md 添加文件到达缓存

  4. git commit -m ‘first’ 提交文件

  5. git log 可以查看所有产生的 commit 记录

  6. git branch查看所有分支

  7. git branch a 创建一个名为a的分支，可以使用git branch查看到刚刚创建的分支

  8. git checkout a 可以从别的分支转换到a的分支上进行开发

  9. git merge 合并其他分支到主master分支，首先要确认在主分支上使用git checkout master.然后执行git merge a 意思就是将a分支的改动合并到master分支上来。（当然有可能有冲突情况）

  10. git branch -d a 命令删除a分支

  11. git branch -D a 命令强制删除a分支，上面一条命令可能会因为a分支还没有合并到masterer无法删除。

  12. git tag v1.0 就代表我在当前代码状态下新建了一个v1.0的标签，输入 git tag 可以查看历史 tag 记录。

  13. git checkout v1.0可以切换到V1.0版本

  14. git push origin master把本地代码推到远程 master 分支。

  15. git pull origin master远程最新的代码更新到本地。

  16. git remote add origin https://github.com/moonoff/MyApi.git建立关联，origin只是一个仓库名字，可以修改为别的。

  17. git push origin master提交到远程仓库

  18. git log –graph –pretty=format:’%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset’ –abbrev-commit –date=relative使log规范化

  19. git config –global alias.lg “log –graph –pretty=format:’%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset’ –abbrev-commit –date=relative”设置别名，可以使用git lg代替第18条命令

  20. git checkout a.md该命令只能撤销还没有 add 进暂存区的文件。

  21. git diff <$id1> <$id2> # 比较两次提交之间的差异

  22. git diff < branch1>..< branch2> # 在两个分支之间比较

  23. git diff –staged # 比较暂存区和版本库差异

  24. git stash把当前分支所有没有 commit 的代码先暂存起来，并且让你看到原来没有修改的样子

  25. git stash list暂存区已经有了一条记录。

  26. git stash apply你会发现你之前的代码全部又回来了，就好像一切都没发生过一样，紧接着你最好需要把暂存区的这次 stash 记录删除，执行：git stash drop

  27. git stash pop可以执行第26条的所有动作。

  28. git stash clear清除所有的暂存区。

  29. rebase 跟 merge 的区别你们可以理解成有两个书架，你需要把两个书架的书整理到一起去，第一种做法是 merge ，比较粗鲁暴力，就直接腾出一块地方把另一个书架的书全部放进去，虽然暴力，但是这种做法你可以知道哪些书是来自另一个书架的；第二种做法就是 rebase ，他会把两个书架的书先进行比较，按照购书的时间来给他重新排序，然后重新放置好，这样做的好处就是合并之后的书架看起来很有逻辑，但是你很难清晰的知道哪些书来自哪个书架。
